# PRODUCT.md — TokenItDown

register: brand

## Product purpose

TokenItDown turns any document or web page into clean, LLM-ready Markdown, then goes past the conversion to deliver visible quality control, RAG-ready output, token economics, and native AI-agent (MCP) access. The conversion engine (Microsoft MarkItDown) is commoditized; the value is the workflow around it: the library, the repair loop, the RAG export, the per-key transparency, and the agent integration, packaged cleanly for both cloud users and self-hosters.

Two deployment targets from one codebase: a multi-tenant cloud SaaS, and a single-command self-hosted stack (`docker compose up`) with all processing local and no data egress.

## Users

- **AI/LLM engineers and agent builders** wiring documents into RAG pipelines and coding agents who are tired of paying tokens for raw HTML and boilerplate.
- **Developers using coding agents** (Claude Code, Cursor, Copilot, Codex, Gemini, Windsurf) who want the agent to read a file or URL directly via MCP, no copy-paste.
- **Self-hosters and privacy-conscious teams** who need conversion to happen on their own infrastructure with no third-party egress.

They are technical, skim-first, and allergic to marketing hype. They trust concrete numbers, real screenshots, code snippets, and open source over adjectives.

## Brand and tone

- **Voice:** professional yet approachable, benefit-led, no hype words. Confident and concrete. Speak to engineers as peers.
- **Identity:** the product carries the TokenItDown wordmark; the company identity is AnHourTec. Open source (MIT).
- **Feeling:** fast, precise, trustworthy, transparent. "Nothing hidden" is a core theme (per-key transparency, open source, self-hostable).

## Strategic principles

- Show, don't claim: lead with the actual product (screenshots, real tools, real token numbers) rather than abstract promises.
- Token economy is the hook: the core benefit is cleaner input for a fraction of the tokens.
- Openness is a feature: MIT license, self-hostable, community-driven. Invite collaboration explicitly.
- Meet developers where they are: MCP install snippets, works with any agent, copy-paste-ready.

## Anti-references

- Generic AI-SaaS landing pages: big gradient hero blob, three identical icon+heading+text cards, "hero metric" stat strip, vague "supercharge your workflow" copy.
- Hype adjectives (revolutionary, seamless, magical, powerful) with nothing concrete behind them.
- Stock illustrations of robots or floating documents. Use the real UI instead.
